# 官方關卡 Goals 語意

來源：`關卡格式資料/Level_*.json` 的 `Goals[]`（`Goal` = 目標類型 ID，`Count` = 數量）。

## Goal ID 對照

| Goal ID | 物件 | Count 代表 | 對玩家直觀？ |
|--------:|------|------------|-------------|
| 12 | 紙箱 Crt | 要消除的**次數**（每箱多 HP 就多下幾次） | 大致 = 盤上箱數（若每箱 1 HP） |
| 13 | 水漥 Puddle | **命中次數**（多在 `Cells` 層，開局盤面常為 0） | 否 — 玩過程中累積 |
| 14 | 礦泉水櫃 | **HP 總和**（每台約 11） | 次數，非「台數」 |
| 15 | 郵戳 Stamp | **蓋章次數**（製造機） | 否 — 遠大於盤上郵戳數 |
| 16 | 木桶 Barrel | instance 消除次數 | 約 = 盤上桶數 |
| 17 | 飲料櫃 | **HP 總和**（每台約 5） | 次數 + 可標台數 |
| 19 | 鮭魚罐 | instance 次數 | ≈ 盤上數量 |
| 20 | 泳池 Pool | instance 次數 | ≈ 盤上 2×2 台數 |
| 21 | 泥巴 Mud | 命中次數（`Cells`） | 否 |
| 26 | 交通錐 | instance 次數 | 多數關 ≈ 盤上數 |

**結論**：官方 goal 多半是「消除命中次數」，不是單純「盤上有幾個物件」。對多 HP 障礙物，玩家看到的是扣血次數，與官方一致。

靜態關卡（僅元素補位、無障礙生成）時，Crt / Barrel / SalmonCan 等 **Count 常等於開局 instance 數**。

## 落下 / 生成機制

| 機制 | 說明 | 關卡數 |
|------|------|--------|
| `FillType: 1` | 頂部補**元素**（幾乎全關都有） | ~100 |
| `Sets.Elements` 含障礙 ID (≥21) | 遊戲中**生成障礙物** | **12 關** |

會生成障礙物的關卡：**37, 38, 39, 40, 46, 58, 60, 66, 70, 80, 93, 95**（例：L37 Sets 含 Barrel Id 32）。

這些關 goal 可大於開局盤面數（例 L37 Barrel goal 103、盤上 53）。
