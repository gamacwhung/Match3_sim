# 消除規則（精簡版）

對照設計概念：**一般 match = 相鄰消除 + 部分原地消除**；**道具觸發 = 主要原地消除**（不向外擴散到鄰格障礙），**例外**見下文。

真值：`match_engine.py`（Python）、`godot_demo/scripts/board/game_board.gd`（Godot）。

---

## 0. 延伸消除（元素，設計文件 B 節）

與「障礙物相鄰消除」不同：多消**同色元素格**。

| 合成形狀 | 延伸規則 |
|----------|----------|
| **三消** | **無**延伸 |
| **橫三 + 正下一格**（T 形 4 格） | 併為 **L/T 群組**，四格一起消 → 合成**炸彈** |
| **五連** | 排列**中央**的上下鄰格同色 |
| **L/T 炸彈** | 中央四鄰同色，若與群組形成 4 連或 2×2 則多消 |
| **2×2 紙飛機** | 方塊旁同色，與方塊兩格共線成 3 連則多消 |

### 紙飛機飛行目標（權重）

- **單獨紙飛機**：選權重最高的障礙物格（目標 +100、HP=1 +1，見 `tile_defs.TRPR_*`）。
- **紙飛機 + 炸彈 / 火箭**：落點依「在該格施放道具後，爆炸範圍內所有障礙物權重加總」選最高（5×5 或整行/列）；2×2 障礙只計一次。

實作：`match_engine.collect_extended_elimination`、`match_finder` T 形短臂合併。

## 1. 一般三消 / L 形 / T 形（MATCH）

每個**被消除的元素格**，向上下左右 4 格檢查障礙物：

- 鄰格**也在同一個 match 群組內** → **不**再算相鄰消（避免重複扣血）。
- 鄰格障礙物 `can_adjacent_elim=true` → 扣 1 血（同一 match tick 內同格只扣一次）。

### L 形範例（你問的 XXX / X）

```
. . .
A A A    ← 橫三連（都在 match 內）
. B .    ← B 是第四格（也在 match 內）
```

消除後相鄰檢查：

| 格 | 會檢查的鄰居方向 | 說明 |
|----|------------------|------|
| 中間 A | 上、下、左、右 | **下** 會打到 B 正下方那格；**右** 若旁邊有郵戳/紙箱會觸發 |
| 下方 B | 上、下、左、右 | **上** 是 match 內 → 跳過；**左、右、下** 仍會觸發相鄰消 |

因此 L 形**應該**讓「突出那一格」四周（不含 match 內部）的障礙物受傷；若只有 match 正上方/正下方有障礙，看起來會像「只有一側有反應」——那是正常幾何結果，不是漏算。

Godot 額外：` _trigger_manufacturers_adjacent_to_cells(cells)`，確保與 match 相鄰的**郵戳**都會蓋章（L21 中間欄常用到）。

## 2. 原地消除（INPLACE）

消除格**同一格**上的層（非 4 鄰）：

| 層 | 例子 | 條件 |
|----|------|------|
| bottom | 水漥 Puddle | `can_inplace_elim` |
| upper | 繩索 Rope | `can_inplace_elim` |

---

## 3. 道具消除（SPECIAL / PLANE）

| 模式 | 行為 |
|------|------|
| `EXPLODE_MODE_SPECIAL` | 只傷**該格**上的障礙物，**不**打 4 鄰 |
| `EXPLODE_MODE_PLANE` | 紙飛機落點：只消落點 1 格 |
| `EXPLODE_MODE_MATCH` | 同一般 match： candy 消除 + `_trigger_obstacle_adjacent` |

適用 SPECIAL 的：條紋整行/列、包裝 3×3、彩球清色、TNT 5×5、道具連鎖內的條紋/包裝/彩球。

---

## 4. 合成「十字 4 鄰」（MATCH，不是 SPECIAL）

下列情況的 **4 鄰格** 必須走 **MATCH**（相鄰消除），否則旁邊郵戳/紙箱不會被打到：

- 2×2 合成時的十字四格
- 紙飛機 + 其他道具合成時的 4 鄰
- 紙飛機**連鎖觸發**時的 4 鄰（`_chain_trigger` SPIRAL）
- 彩球+火箭等合成點的 `get_cross_targets`

```
  .
A B    ← 2×2 四角消除後，每格仍對自己的 4 鄰做 adj
. .
```

---

## 5. 製造機（郵戳 Stamp）

- `category=manufacturer`，**不會被消除**。
- 相鄰 match 時：GOAL +1（每 match tick 每枚郵戳最多 +1），播放蓋章。
- 與一般障礙物不同：扣的是**目標次數**，不是 HP。

---

## 6. 櫃子類（Chiller）

| 類型 | 相鄰 match | 道具 | 備註 |
|------|------------|------|------|
| WaterChiller | 每 match 每 instance 最多 -1 | **開門後**範圍內每格各 -1（不去重） | 關門仍每道具啟動 -1 |
| BeverageChiller | 須**對色**；殺對色瓶；match 內 instance 去重 | 每次啟動 instance 最多 -1 | 關門：對色開門不殺瓶 |

HUD 顯示：`N 次` + `（M 台 2×2）`（instance 計數）。

## 7. 常見誤解排查

| 現象 | 原因 |
|------|------|
| L 形只燒一側 | 另一側沒有 `can_adjacent_elim` 的障礙物 |
| 郵戳沒動 | 曾把 Stamp 映射成 ice（已修：loader 用 `manufacturer`） |
| 2×2 旁郵戳沒蓋章 | 4 鄰曾用 SPECIAL（已改 MATCH） |
| TNT 不燒隔壁 | 設計如此（SPECIAL 只打格內） |
